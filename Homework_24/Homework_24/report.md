- Django ORM без оптимізації генерує багато SQL-запитів; `select_related` і `prefetch_related` значно знижують їх кількість та прискорюють виконання
- Кешування через Django cache (Redis або Memcached) дозволяє швидко віддавати сторінки без додаткових запитів до бази
- Django Debug Toolbar допомагає знаходити вузькі місця у запитах і шаблонах.
- Raw SQL може знадобитися для складних запитів; обов'язково використовувати параметризовані запити для захисту від SQL-ін'єкцій.
- Celery дозволяє виконувати довгі задачі (імпорт CSV, відправка email) асинхронно, не блокуючи веб-запити
- MongoDB через mongoengine можна підключати паралельно до реляційної бази і зберігати/читати дані незалежно
