## 1. Дайте визначення наступним термінам: XSS, SQL injection, CSRF, clickjacking, параметризовані запити, middleware, salt, hashing.
* XSS - (cross-site scripting) це коли в сторінку вбудовують шкідливий JS
* SQL injection - вставка стороннього SQL коду у запит, щоб змінити або вкрасти дані з БД
* CSRF - (cross-site request forgery) атака при якій користувача обманом змушують виконати не бажаний запит від його імені
* Clikjacking - коли сторінку ховають у айфрейм щоб людина натиснула на елемент, якого не бачить
* Параметризовані запити - це запити з параметрами в плейсхолдерах. Запобігають SQL ін'єкціям
* Middleware - проміжна логіка що (в даному випадку) перехоплює і обробляє запити\відповіді між сервером і клієнтом
* Salt - випадковий фрагмент тексту, якиий додається до пароля перед хешуванням щоб зробити його унікальнім
* Hashing - односторонній процес перетворення даних у рядок фіксованої довжини

## 2. Опишіть основні вразливості, до яких може бути схильний додаток на Django, та способи їх запобігання
* XSS -> Django автоматично екранує HTML
* CSRF -> використання CSRF-токенів
* SQL injection -> використання ORM або параметризованих запитів
* Clickjacking -> встановлення заголовка X-Frame-Options
* Brute Force -> застосування лімітів на кількість запитів
* Витік даних -> використання HTTPS

## 3. Як захистити конфіденційні дані користувачів у Django-додатку? Які механізми шифрування можна використовувати?
* Використовувати хешування паролів за допомогою PBKDF2 та Аrgon2
* передавати дані лише через HTTPS
* зберігати ключі та інші секретні дані окремо від коду у змінних оточення

## 4. Які способи аутентифікації та авторизації користувачів передбачені в Django? Опишіть їх переваги та недоліки
Для цього використовується вбудований модуль ```django.contrib.auth```, сесійна аутентифікація, використання токенів

## 5. Як забезпечити безпеку сесій користувачів? Які загрози можуть виникнути і як їх уникнути?
Треба використовувати secure cookies (SESSION_COOKIE_SECURE) та CSRF захист, скорочений час життя сесії
Загроза - перехоплення cookies - треба використовувати HTTPS

## 6. Які заходи можна вжити для захисту Django-додатка від DDoS атак?
* додати тротлінг через DRF
* використовувати CDN
* вмикати кешування
* налаштувати захист самого сервера

## 7. Як захистити форми від підробки міжсайтових запитів (CSRF)?
* Додавати {% csrf_token %} у кожну форму, що надсилає POST.
* Не вимикати CSRF middleware.
* Перевіряти токен на сервері при кожному запиті POST.
